# 双五句法

根据文体选择不同扩写法。

## A. 分析型五句法

适合技术文章、议论文、复盘、科技评论。

```text
1. 事实句：发生了什么？
2. 判断句：这说明什么？
3. 比较句：和过去 / 旧方案 / 其他对象相比，有什么变化？
4. 意义句：这对工程、行业、用户或个人有什么影响？
5. 边界句：这个判断在哪些条件下成立？不能扩大到哪里？
```

示例：

```text
事实句：Safari 和 Firefox 稳定版开始支持 WebGPU。
判断句：这说明 WebGPU 正在从 Chromium 生态的前沿能力，转向主流浏览器共同落地的 Web 平台能力。
比较句：和 WebGL 主要承担图形渲染不同，WebGPU 同时面向图形渲染和通用 GPU 计算。
意义句：对工程项目来说，3D、可视化和轻量本地 AI 推理可以更认真地把 WebGPU 作为主路径设计。
边界句：不过，它仍然不能被视为全平台无差异 API，实际使用时仍需要 feature detection 和 fallback。
```

## B. 血肉型五句法

适合散文、评论、复盘、情绪表达。

```text
1. 观点句：这一段想说什么？
2. 场景句：这个问题在哪里出现？
3. 机制句：它为什么会这样？
4. 例子句：有没有具体例子？
5. 克制句：什么时候不成立，或者我该如何收住？
```

示例：

```text
观点句：我不是没有想法，而是想法太快，文字跟不上。
场景句：每次打开笔记软件，我脑子里同时冒出项目、文章、学习计划和旧资料整理。
机制句：这些想法没有经过排序，就会全部挤到同一页上，最后变成一堆标题。
例子句：我明明想写一篇 WebGPU 的文章，最后却只列出了年份、浏览器和框架名称。
克制句：问题不在于想法太多，而在于我还没有把想法压成一条能带读者走下去的判断线。
```
