# 文体模板

根据文章类型灵活选择，不要机械套用。

## 技术文章

```text
背景 → 现象 → 核心判断 → 和旧方案比较 → 机制拆解 → 工程意义 → 风险/边界 → 方法论收束
```

重点检查：

- 有没有中心判断；
- 有没有和旧技术比较；
- 有没有机制解释；
- 有没有工程意义；
- 有没有边界和 fallback；
- 有没有避免过猛技术判断。

## 技术日志

```text
事实记录 → 我看到的变化 → 和过去相比 → 我的判断 → 对工程的意义 → 暂时的边界
```

重点检查：

- 是否只是资料清单；
- 有没有"这说明什么"；
- 有没有"我的判断是"；
- 有没有"但边界是"。

## 议论文

```text
提出问题 → 核心判断 → 判断成立的原因 → 具体例子 → 处理反对意见 → 限定边界 → 现实意义
```

重点检查：

- 观点是否明确；
- 论据是否支撑观点；
- 有没有反方意见；
- 有没有边界；
- 有没有现实落点。

## 散文

```text
具体场景 → 细微观察 → 动作/细节 → 感受变化 → 联想 → 克制的判断 → 留白
```

重点检查：

- 是否一上来讲大道理；
- 是否缺少场景和动作；
- 情绪是否有层次；
- 结尾是否留有余味。

## 复盘文章

```text
背景 → 过程 → 关键决策 → 当时误判 → 后来验证 → 教训 → 方法论 → 下次行动
```

重点检查：

- 是否只有结论没有过程；
- 是否说明当时为什么这么做；
- 是否有验证证据；
- 是否有可迁移的方法论。

## 科技论文式文章

```text
问题定义 → 背景/相关工作 → 方法 → 实验/案例 → 结果 → 讨论 → 局限 → 结论
```

重点检查：

- 问题是否定义清楚；
- 方法和结果是否区分；
- 讨论是否有边界；
- 结论是否没有超出证据。
