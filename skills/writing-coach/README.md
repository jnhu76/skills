# Writing Coach

## 这个 skill 是干什么的？

系统训练用户写出有细节、有逻辑、有情绪层次的文章。诊断大纲感过强、逻辑跳跃、情绪过满、细节不足等问题，使用“五句扩写法”和文体适配模板进行训练、批改和反馈。不代写，先诊断，再扩写，最后布置训练任务。

## 什么时候用？

适合：

- 写完一段文字或大纲，觉得太像提纲、缺乏细节，需要诊断和改写建议
- 情绪写得太满，想把情绪判断改成有层次的观察与表达
- 逻辑跳跃，需要补全中间推理链（A→B→C）
- 想针对特定文体（技术文章、议论文、散文、复盘、科技论文）学习结构模板
- 需要获得具体的训练任务来持续提升写作能力

## 什么时候不用？

不适合：

- 只需要语法校对、拼写检查或简单润色，而不需要写作训练
- 只需要翻译或改写他人文字，而不是提升自己的原创表达
- 想快速拿到成稿，完全不想学习写作方法（若明确要求代写，skill 仍会在完成后指出扩写方法）

## 调用 Prompt

```markdown
请使用 `writing-coach` skill 完成下面任务。

我的目标是：

```text
<目标，例如：把这段大纲扩展成有细节的技术复盘 / 诊断这段散文的问题 / 给这个议论文观点补逻辑链>
```

我提供的材料是：

```text
<用户写的文字、大纲、观点或主题>
```

请输出：

1. 判断文体和当前状态
2. 诊断主要问题（3-5 条）
3. 最需要补充的层面
4. 逐段改写建议（含 Before/After 示范）
5. 扩写版本
6. 一个可执行的训练任务
```

## 输入材料建议

最好提供：

```text
- 你写的原始文字、大纲或草稿（越完整越好）
- 目标文体（技术文章 / 议论文 / 散文 / 复盘 / 科技论文 / 不确定）
- 写作目的和读者对象（如有）
- 你最不满意的具体段落或句子（如有）
```

## 输出格式

应该包含：

```text
一、这段文字目前像什么？
二、最大问题是什么？
三、最应该补什么？
四、逐段改写建议（原句 / 问题 / 改法 / 示范）
五、扩写版本
六、训练任务
```
