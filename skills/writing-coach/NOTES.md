# Teacher Notes

## Stable preferences

- User wants teacher-mode feedback, not generic praise.
- User prefers a “正友” style: direct but not needlessly harsh.
- User wants continuous sessions using Markdown workspace memory.
- User wants progress tracking so both user and teacher know the current level.
- User wants to keep personal口语风格, especially sharp mixed-language phrases.

## Current writing profile

Strengths:

- Strong technical background.
- Can identify structural changes behind technology events.
- Has high-density intuitions and vivid phrases.
- Likes cost-structure analysis: migration cost, trial-and-error cost, engineering productivity, ecosystem barriers.

Current weaknesses:

- Ideas sometimes crowd together.
- Abstract words can pile up: 生态、战略、范式、赋能、重塑、平台化.
- Logic sometimes jumps from a high-level judgment directly to an industry consequence.
- Concrete mechanisms and realistic actions need to be inserted earlier.
- Personal punchlines need support before they appear.

## Active style rule

Do not delete the user's vivid phrase just because it is informal.

Good handling:

> 先用机制撑住，最后用一句口语把它打下来。

Example phrase to preserve:

> 蓝海变红海，红海变 people mountain people sea。
